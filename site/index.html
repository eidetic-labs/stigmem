What is Stigmem?

A federated knowledge fabric

Stigmem provides a shared, persistent substrate where typed facts flow between agents, tools, and organizations with full provenance.

🧠

Immutable Facts

Every fact is a typed record with entity, relation, value, source, timestamp, confidence, and scope. Full provenance on every write.

🌐

Federated by Design

Nodes peer via Ed25519-signed handshakes. Facts replicate across scope boundaries under explicit permission. No central coordinator required.

⏳

Natural Decay

Facts have expiry times and confidence scores. The substrate stays fresh like pheromone trails that fade when no longer reinforced.

⚡

Contradictions as Data

Conflicting facts from different sources are surfaced as first-class records, never silently overwritten. Resolution is explicit.

Problems We Solve

AI coordination is broken

Multi-agent systems today lack a shared cognitive layer. Stigmem fixes that.

1

Knowledge Silos

Agents accumulate knowledge in isolated contexts. When a session ends, the knowledge dies. Stigmem persists facts across sessions, tools, and organizations.

2

No Shared Context

Multiple agents working on the same problem can't see each other's discoveries. Stigmem provides a shared environment where traces guide future behavior.

3

Silent Conflicts

When two systems disagree, one usually wins silently. Stigmem surfaces contradictions as first-class records with explicit resolution paths.

4

Centralization Risk

Vendor lock-in for agent memory creates single points of failure. Stigmem is federated, open-source, and designed for multi-party deployment.

5

Stale State

Knowledge bases accumulate outdated facts with no mechanism for expiry. Stigmem's decay model ensures facts that aren't reinforced naturally fade.

6

Missing Provenance

You can't trust a fact you can't trace. Every Stigmem fact carries its source, timestamp, and confidence — audit trails are built in.

How it Works

Stigmergy meets memory

Like ant colonies coordinating through environmental traces, agents write facts to a shared substrate that guides future behavior.

The Fact Shape

Every piece of knowledge in Stigmem is an immutable, typed record:

{
  "entity": "stigmem://acme-corp/user/jane",
  "relation": "prefers_language",
  "value": "Python",
  "source": "agent:onboarding-bot",
  "confidence": 0.92,
  "valid_until": "2026-12-01T00:00:00Z",
  "scope": "acme-corp/engineering"
}

Hybrid Logical Clocks — causality-preserving timestamps that work across distributed nodes

Scope-based access — facts are visible only within their declared scope boundary

Pull replication — federated nodes sync facts under explicit peering agreements

Adapter ecosystem — MCP, Paperclip, and OpenClaw integrations ship out of the box

Why We're Building This

Intelligence should compound

Today's AI systems are brilliant but amnesiac. Every agent session starts from scratch. Every tool call is stateless. Knowledge discovered by one agent is invisible to the next.

We believe multi-agent intelligence should compound, not reset. When an agent learns something, that knowledge should persist, travel to where it's needed, and decay when it's no longer true.

The name says it all: Stigmergy (coordination through environmental traces, like ant colonies) + Memory (persistent facts with natural decay). No central controller, no vendor lock-in — just a shared cognitive layer that any agent can read from and write to.

Stigmem is the substrate that makes multi-agent systems remember — across sessions, across platforms, across organizations. Open source, federated, and built for a world where AI agents are everywhere.
