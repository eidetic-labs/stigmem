**Last updated:** 2026-05-21

## Release Intent

`v0.9.0a1` was the first canonical Stigmem build after the withdrawn v1.0
announcement. Its purpose was to reset the public version line, correct the
project maturity claim, publish installable first-build artifacts, and establish
the alpha posture that later release lines build on.

The release deliberately did not claim stable production readiness. It made the
project installable and inspectable while preserving the future hardening,
plugin extraction, and operator-validation work as alpha/beta horizon work.

## Detailed Scope

### Version Reset and First Public Artifacts

**Objective:** establish `v0.9.0a1` as the first canonical public build.

**Included changes:**
- Reset public prose, package metadata, and release surfaces to the alpha line.
- Publish `stigmem`, `stigmem-py`, `stigmem-node`, `stigmem-openclaw`, and
  `@eidetic-labs/stigmem-ts` on the alpha version line.
- Record earlier version markers as internal development checkpoints.

**Excluded / deferred:**
- No stable wire-format guarantee.
- No retroactive replacement of prior internal checkpoint content.

**Evidence required:**
- Package/version surface audit.
- Changelog entry describing the reset and artifact set.
- Security posture updated for the supported alpha line.

**References:**
- `CHANGELOG.md`
- `release/version-surfaces.yaml`
- `SECURITY.md`

### Public Retraction and Posture Calibration

**Objective:** correct the public maturity claim and document safe adoption
boundaries.

**Included changes:**
- Public retraction and README explanation.
- Alpha stability banner.
- Supported deployment guidance pointing adopters to safe local/demo paths.
- Security posture reset away from the withdrawn v1.0 claim.

**Excluded / deferred:**
- Production federation across organizational boundaries.
- External operator soak.
- Hardened-core security controls that remained open.

**Evidence required:**
- Retraction URL preserved.
- README, `LIMITATIONS.md`, `SECURITY.md`, and roadmap aligned on alpha posture.

**References:**
- `README.md`
- `LIMITATIONS.md`
- `SECURITY.md`
- `archive/devto-stigmem-v0.9.0a1-retraction.md`

### Default-Surface Boundary

**Objective:** make the default install match the critical-path scope while
keeping experimental work available for continued development.

**Included changes:**
- Move deferred feature material into `experimental/` where applicable.
- Keep CIDs as core per the later ADR-017 boundary.
- Identify non-default features as experimental or deferred.

**Excluded / deferred:**
- Multi-tenant, RTBF tombstones, time-travel queries, lazy instruction
  discovery, advanced memory-garden ACL, source attestation, dashboard, and
  other cross-cutting features as supported default behavior.

**Evidence required:**
- Feature matrix and docs state supported vs. experimental posture.
- Repo layout reflects ADR-009/ADR-011 direction.

**References:**
- `README.md`
- `docs/internal/feature-tracker.md`
- `docs/docs/concepts/features.md`

## Security and Risk Posture

`v0.9.0a1` was a preview alpha. It did not close the hardened-core security
requirements for stable production federation. The security posture documented
which dependency/code-level mitigations existed and which threat-model controls
remained future work.

Residual risk included mTLS-default federation, persistent audit retention,
per-principal rate limits, bounded HLC skew enforcement, capability-level
instruction validation, and storage immutability.
