# v0.9.0a6 Release Roadmap

**Status:** active
**Release owner:** maintainer
**Milestone:** `v0.9.0a6`
**Tag / release:** not created
**Last updated:** 2026-05-23

## Release Intent

`v0.9.0a6` validates the Memory Garden advanced ACL extraction in the alpha
series. The historical roadmap and PR4e work established the boundary: basic
garden CRUD, admin/writer/reader membership, direct `garden_id` write
validation, direct `garden_id` read validation, and scope-mismatch rejection
remain core; advanced cross-surface ACL behavior belongs to the opt-in
`stigmem-plugin-memory-garden-acl` boundary.

This release is a validation and artifact-refresh horizon. It does not remove
the core garden primitive, graduate Memory Garden ACL into the supported
surface, open beta/RC/GA, or publish standalone plugin artifacts unless issue
#298 is explicitly moved into this milestone.

## Security and Risk Posture

`v0.9.0a6` remains an alpha release. Memory Garden advanced ACL is
experimental and opt-in. The main security invariant is boundary honesty:
default installs must keep basic garden protections, advanced ACL behavior must
not activate without plugin registration and operator enablement, and any
cross-surface behavior that cannot safely move through the current hook surface
must be retained in core or documented as a follow-up gap.

R-21 handling is security-sensitive. The a6 release must state whether Memory
Garden advanced ACL implements, supports, or only coexists with same-session
read/write graph isolation. It must not claim R-21 closure unless the required
session, graph, and audit evidence exists.

Security publication policy remains unchanged: Critical/High disclosures use
GHSA where applicable; Medium/Low findings remain in `SECURITY.md` unless a
documented risk-profile, reporter-coordination, or downstream-compliance
carve-out applies.
