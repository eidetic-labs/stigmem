# v0.9.0a2 Release Roadmap

**Status:** historical
**Release owner:** maintainer
**Milestone:** historical; pre-standard milestone discipline
**Tag / release:** `v0.9.0a2`
**Last updated:** 2026-05-21

## Release Intent

`v0.9.0a2` was the first alpha artifact refresh after the `v0.9.0a1` posture
reset. Its purpose was to publish the source-side corrections that had landed
after the first build, close the Phase A extraction evidence, correct
OpenClaw/ClawHub alpha framing, and publish the security hardening/advisory
evidence for the supported alpha line.

The release did not open beta, RC, or GA. It remained part of the alpha series.

## Scope Summary

| Area | In scope | Out of scope | Canonical references |
| --- | --- | --- | --- |
| Artifact refresh | Core packages, SDKs, OpenClaw adapter, GHCR, docs, release evidence | Plugin artifact publication evidence | `CHANGELOG.md`, release evidence docs |
| Plugin extraction | Complete source extraction for the six ADR-011 cross-cutting features | Declaring extracted features stable or default-on | `ROADMAP.md`, feature matrix, compatibility matrix |
| Phase A exit | Validate extraction set, default-install behavior, plugin docs/trust posture | 30-day external operator soak | `ROADMAP.md`, tests, docs |
| OpenClaw/ClawHub framing | Correct alpha/evaluation-only language and stale package guidance | Retroactively changing `v0.9.0a1` artifacts | `CHANGELOG.md`, OpenClaw docs |
| Security publication | Critical/High via GHSA; Medium/Low in `SECURITY.md`; evidence registry | Treating Medium/Low dispositions as future publication work | `SECURITY.md`, evidence registry |
| Repo/docs hygiene | Docs/spec cleanup, dogfood disposition, evidence maintenance | Removing strategic roadmap detail | `docs/internal/` |

## Detailed Scope

### Artifact Refresh

**Objective:** publish the next alpha artifact set from the corrected source
state.

**Included changes:**
- Advance Python, npm, FastAPI/OpenAPI, conformance, plugin fallback, Docker
  Compose, and release-documentation surfaces to the alpha.2 line.
- Capture release evidence for core packages, SDKs, OpenClaw adapter, docs, and
  GHCR surfaces.
- Preserve immutable `v0.9.0a1` artifacts instead of editing them in place.

**Excluded / deferred:**
- Plugin package publication evidence, because no released plugin artifacts
  existed to attest.
- Any beta/RC/GA milestone opening.

**Evidence required:**
- Version-surface alignment.
- Release verification commands.
- Changelog and release evidence.

**References:**
- `CHANGELOG.md`
- `docs/docs/operators/release-verification.md`
- `release/version-surfaces.yaml`

### Plugin Extraction and Phase A Exit

**Objective:** complete source-side extraction of cross-cutting experimental
features while preserving default-install behavior.

**Included changes:**
- Source packages under `experimental/` for lazy instruction discovery,
  time-travel, RTBF tombstones, memory-garden advanced ACL, source attestation,
  and multi-tenant isolation.
- Default install remains the critical-path behavior.
- Plugin-loaded tests cover feature boundaries.
- Signed/package artifact launch evidence remains tracked separately.

**Excluded / deferred:**
- Declaring extracted features stable or supported.
- Moving `Spec-X*` features into `Spec-NN`.
- ADR-008 graduation gates.

**Evidence required:**
- Feature-boundary tests.
- Default-install no-registration behavior.
- Docs/spec status metadata.
- Follow-up issue for plugin artifact evidence.

**References:**
- `ROADMAP.md`
- `docs/docs/concepts/features.md`
- `docs/compatibility-matrix.yaml`

### OpenClaw / ClawHub Alpha Framing

**Objective:** correct public language so OpenClaw is described as alpha /
evaluation-only.

**Included changes:**
- Remove production-integration framing.
- Correct stale dependency ranges and package guidance.
- Preserve historical package immutability.
- Keep remaining hardening visible in the alpha/future hardened-core path.

**Excluded / deferred:**
- Retroactive edits to historical `v0.9.0a1` package contents.
- Claiming OpenClaw audit findings are all closed by the first build.

**Evidence required:**
- Source/docs copy reflects alpha/evaluation posture.
- Changelog notes the framing correction.

**References:**
- `CHANGELOG.md`
- OpenClaw adapter docs and compatibility pages.

### Security Publication and Evidence

**Objective:** publish the `v0.9.0a2` security posture and advisory evidence.

**Included changes:**
- Critical and High findings disclosed through GitHub Security Advisories.
- Medium and Low findings documented in `SECURITY.md` and release docs per
  policy.
- Dated internal evidence registry records audit finding lineage, PRs, paths,
  tests, and dispositions.
- R-24/R-25/R-26 registered as mitigated architectural risks where applicable.

**Excluded / deferred:**
- Framing already-documented Medium/Low dispositions as future publication work.
- Adding every audit finding to the threat model.

**Evidence required:**
- `SECURITY.md` disposition index.
- `docs/internal/security-evidence-registry-2026-05-17.md`.
- `spec/security/evidence-registry.json` validation.

**References:**
- `SECURITY.md`
- `docs/internal/security-evidence-registry-2026-05-17.md`
- `spec/security/threat-model.md`
- `spec/security/evidence-registry.json`

## Security and Risk Posture

`v0.9.0a2` includes security hardening and advisory publication for the
supported alpha line. Critical and High findings are disclosed through GHSAs.
Medium and Low findings are captured in `SECURITY.md` and release docs unless
their risk profile changes.

The release remains a preview alpha. Hardened-core production federation
controls and external-operator validation remain future work.
