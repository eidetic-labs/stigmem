**Audience:** Node operators, SDK consumers, and protocol implementers.

## Summary

Stigmem v2.0 promoted seven specification sections (§19–§25) from draft to normative, completing the protocol's federation security, semantic recall, data protection, and content-integrity layers across eight development phases (8–14).

## Known caveats

<ol className="stigmem-steps">
<li><strong>Embedding model lock-in.</strong> Changing <code>STIGMEM_EMBED_DIMENSIONS</code> after indexing requires a full re-index.</li>
<li><strong>Mixed-version federation.</strong> v1.x and v2.0 nodes cannot federate. All nodes must upgrade together.</li>
<li><strong>CID backfill duration.</strong> Large fact stores (&gt;1M facts) may take hours. Backfill runs concurrently with live writes.</li>
<li><strong>Tombstone propagation latency.</strong> Up to 5 minutes via polling; ~60 seconds via the <code>tombstone_new</code> subscription channel.</li>
<li><strong>Transparency log dependency.</strong> In <code>trust_mode: strict</code>, federation connections fail if the log is unreachable.</li>
</ol>
