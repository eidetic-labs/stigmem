# Stigmem Documentation Site

Docusaurus 3.x documentation for the [Stigmem protocol](../README.md).

## Prerequisites

- Node.js ≥ 18
- npm ≥ 9

## Commands

| Command | Description |
|---------|-------------|
| `npm install` | Install dependencies |
| `npm run start` | Start local dev server (hot reload) at `http://localhost:3000` |
| `npm run build` | Build static site to `build/` |
| `npm run serve` | Serve the production build locally |
| `npm run gen-api-docs` | Regenerate interactive API reference from OpenAPI schema |
| `npm run clean-api-docs` | Delete generated API reference MDX files |
| `npm run clear` | Clear Docusaurus cache |

## Local preview

```bash
npm install
npm run start
```

## Building for production

```bash
npm install
npm run build
# Optionally verify the build locally:
npm run serve
```

## Regenerating the API reference

The interactive API reference is generated from the FastAPI OpenAPI schema. To refresh it:

```bash
# Terminal 1 — start the reference node
cd ../node
uv run python -m stigmem_node    # serves at http://localhost:8000

# Terminal 2 — regenerate API docs
npm run gen-api-docs             # reads from openapi/stigmem.json by default
npm run start
```

The plugin reads `openapi/stigmem.json` (committed) by default, so the build works without a running node. Update `openapi/stigmem.json` from a live node when the schema changes:

```bash
curl http://localhost:8000/openapi.json > openapi/stigmem.json
npm run gen-api-docs
```

## Required frontmatter

Every doc page (except auto-generated API reference) **must** include these frontmatter fields. The build will fail if `audience` is missing.

### `audience` (required)

Declares the primary reader for the page. Rendered as a colored badge next to the page title.

| Value | When to use |
|-------|-------------|
| `Operator` | Deployment, security, observability, runbooks, CLI reference |
| `Integrator` | Guides, SDKs, connectors, tutorials — building on top of Stigmem |
| `Spec` | Protocol specification, architecture docs |
| `Curious` | Getting started, concepts, glossary, community pages |

```yaml
---
title: mTLS Federation Transport
audience: Operator
---
```

### `status` (optional)

Renders a prominent admonition banner at the top of the page linking to the [Features status definitions](/docs/learn/features#how-to-read-this-page). Only add this to pages describing non-stable features.

| Value | Meaning |
|-------|---------|
| `Experimental` | Behind a flag, breaking changes expected |
| `Beta` | Feature-flagged or early adopters, minor breaking changes possible |
| `Planned` | Spec draft exists, not yet implemented |

```yaml
---
title: OIDC / SSO Integration
audience: Integrator
status: Beta
---
```

Stable features do not need a `status` field.

## Adding pages

### New guide or concept page

Create a `.md` file in the appropriate `docs/` subdirectory:

```
docs/guides/my-guide.md        # → /docs/guides/my-guide
docs/spec/federation.md        # → /docs/spec/federation
```

Docusaurus auto-generates the sidebar entry via `type: autogenerated` in `sidebars.js`.

### New section

1. Create a directory under `docs/` (e.g., `docs/tutorials/`)
2. Add an `index.md` as the section landing page
3. Add a new entry to `sidebars.js`:
   ```js
   tutorialsSidebar: [{ type: 'autogenerated', dirName: 'tutorials' }],
   ```
4. Add a navbar item in `docusaurus.config.js`.

### Versioned docs

Single-version mode through the v0.9.0aN alpha series. The first versioned docs snapshot will be cut at v1.0.0 GA per [ADR-001](../docs/adr/001-versioning.md) and `CONTRIBUTING.md` §"Docs versioning snapshot procedure". Until then, `docs/` (current) is the only authoritative source and is published at the unprefixed `/docs/` path on docs.stigmem.dev.

| Label | Path prefix | Source | Purpose |
|-------|-------------|--------|---------|
| `v0.9.0a1` (current) | `/docs/` | `docs/` | Active alpha series; unversioned URLs resolve here |
| (none yet) | — | `versioned_docs/version-v1.0.0/` | Will be cut at v1.0.0 GA |
| `version-v1.1` (legacy snapshot) | — | `versioned_docs/version-v1.1/` | Pre-reset historical record; not built into the live site |
| `version-pre-reset` (legacy snapshot) | — | `versioned_docs/version-pre-reset/` | Pre-reset historical record; not built into the live site |

#### Creating a new version snapshot

When the current docs are ready for a stable release:

```bash
# 1. Ensure the build is clean
npm run build

# 2. Snapshot current docs/ as the new version
npx docusaurus docs:version v1.0.0

# 3. Update docusaurus.config.js:
#    - Set lastVersion to the new version label
#    - Add the new version entry under versions
#    - Relabel current as the next draft after the stable release

# 4. Verify
npm run build
```

This copies `docs/` to `versioned_docs/version-<label>/` and updates `versions.json`.

## Plugin configuration

| Plugin | Purpose | Config key |
|--------|---------|------------|
| `docusaurus-plugin-openapi-docs` | Interactive API reference from OpenAPI schema | `plugins[0]` in `docusaurus.config.js` |
| `@easyops-cn/docusaurus-search-local` | Local full-text search (no Algolia needed) | `plugins[1]` |
| `@docusaurus/theme-mermaid` | Mermaid diagram support in MDX | `themes[2]` |
| `@docusaurus/theme-live-codeblock` | Runnable code examples | `themes[1]` |

## Directory structure

```
stigmem/docs/
├── docusaurus.config.js        # Site config, plugins, navbar
├── sidebars.js                 # Sidebar definitions (learn/build/operate/reference/community)
├── versions.json               # Published versions list [v1.1, pre-reset]
├── openapi/
│   └── stigmem.json            # Local OpenAPI spec (committed)
├── plugins/
│   ├── remark-glossary-link.js # Auto-links glossary terms
│   └── validate-audience.js    # Build-time audience frontmatter validation
├── docs/                       # Current docs (v0.9.0aN alpha series)
│   ├── learn/                  # Concepts, quickstart, how-it-works
│   ├── build/                  # Guides, tutorials, connectors, SDKs
│   ├── operate/                # Deployment, backends, security, observability
│   ├── reference/              # API (generated), spec, architecture, CLI, glossary
│   └── community/              # Security disclosure, project resources
├── versioned_docs/
│   ├── version-v1.1/           # Stable v1.1 snapshot
│   └── version-pre-reset/           # Legacy pre-reset snapshot
├── versioned_sidebars/
│   ├── version-v1.1-sidebars.json
│   └── version-pre-reset-sidebars.json
└── src/
    ├── components/
    │   ├── AudienceBadge/      # Audience pill badge (from frontmatter)
    │   └── FeatureStatus/      # Feature status admonition (from frontmatter)
    ├── theme/
    │   └── DocItem/Content/    # Swizzled to inject badge + status
    ├── css/custom.css
    └── pages/index.tsx         # Landing page
```
