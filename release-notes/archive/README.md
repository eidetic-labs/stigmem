# release-notes/archive — preserved retracted release notes

This directory preserves release-notes documents from version markers that labeled internal development checkpoints rather than publicly-shipped releases. Per [ADR-001](../../docs/adr/001-versioning.md): the canonical version line of stigmem begins at `v0.9.0a1` (2026-05-09).

Each archived release-note file retains a retraction header at the top noting the retraction context and pointing at the v0.9.0a1 reset.

The archive exists per master-checklist §4.3a binding preservation principle: external links to retracted release notes should not 404; the documents that announced these versions are preserved here for historical record.

## How to read these notes

- **They describe state that was announced but not actually shipped.** Do not cite them as evidence of behavior in v0.9.0a1.
- The canonical changelog is `CHANGELOG.md` at repo root.
- The current roadmap is `ROADMAP.md` at repo root.
